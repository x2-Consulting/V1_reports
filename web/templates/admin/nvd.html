Sync Actions
Full Sync

Downloads all ~260k CVEs from NIST NVD and populates the local cache. This takes 3–5 minutes and makes approximately 130 API requests.

Sync Recent (7 days)

Downloads only CVEs added or modified in the last 7 days. Run this daily to keep the cache current without a full re-download.

How the Cache Works
  • When a report is generated, the NVD enrichment step checks the local cache first. If a CVE is found and was cached within the last 30 days, the local data is used immediately with no outbound API call.
  • If a CVE is not in the cache (or is stale), the live NVD API is called, and the result is stored in the cache for future use.
  • Full sync pre-populates the entire database (~260k CVEs). After a full sync, nearly all report generation runs entirely offline.
  • Incremental sync keeps the cache current by pulling only recently modified CVEs. Run it daily or weekly via a scheduled task.
