How Patches Are Prioritised

The report groups every CVE in the CSV by the patch that fixes it, then sorts patches so the most valuable actions appear first:

  1. Most CVEs fixed — a single patch that resolves 20 vulnerabilities appears before one that fixes 2.
  2. Highest CVSS score as a tiebreaker within the same CVE count.

Patch grouping priority: Microsoft KB article → Vendor Patch ID → Security Advisory → Vendor + Product + Version → ungrouped.
